# CSU校园网自动登录

本项目包含用于中南大学（CSU）校园网认证页面的 Tampermonkey 用户脚本及配套的 Windows 批处理文件。结合两者可实现系统开机或断网重连时的全自动认证。

与基于 `requests` 或 `Selenium` 的 Python 自动登录脚本相比，本方案（浏览器脚本 + 系统批处理）具备以下核心优势：

* **零隐私风险**：依托浏览器原生的密码自动填充功能，脚本不存储、不流转任何明文账号密码。
* **极简部署**：完全基于系统原生批处理与基础浏览器扩展，无需配置 Python 环境或安装第三方依赖。
* **免疫反爬机制**：直接运行于浏览器 DOM 层，天然规避校园网登录页面的复杂参数加密或动态反爬策略更新。
* **低资源消耗**：无需拉起独立的 WebDriver 进程，后台网络监测与页面点击的系统开销极低。

## 系统机制与前提

本方案的运行机制分为网络状态监测与自动执行登录两部分。

1. **使用前提**：脚本不包含账号密码的存储与填写逻辑。部署前需确保浏览器已配置校园网账号和密码的自动填充功能。
2. **状态监测**：通过本地批处理文件检测网络连通性，若检测为断网或未认证状态，系统将自动唤起浏览器并跳转至认证页。
3. **自动登录**：浏览器加载认证页后，Tampermonkey 脚本接管操作，检测页面元素并自动触发登录按钮。

## 功能特点

* **全链路自动化**：支持结合 Windows 启动项，实现开机至联网全流程自动化。
* **轻量安全**：基于原生 JavaScript 开发，无第三方依赖，不获取或传输用户隐私数据。
* **异步兼容**：内置轮询监测机制（间隔500ms，最大时限15秒），适应网络波动导致的DOM元素延迟加载。
* **时序控制**：识别目标节点后延迟1秒执行点击操作，确保浏览器前置完成凭据填充。
* **状态反馈**：集成原生 Toast 组件，实时显示脚本运行状态（等待、就绪、超时）。

## 部署指引

### 1. 部署用户脚本

1. 安装 Tampermonkey 浏览器扩展程序。
2. 在 Tampermonkey 管理界面添加新脚本，将 JavaScript 源码粘贴并保存。
3. 验证浏览器已为 `https://portal.csu.edu.cn/` 启用凭据自动填充。

### 2. 部署系统层网络监测

配套的批处理文件（`.bat`）通过向公共 DNS 发送 ICMP 请求以验证网络状态，请求失败则调用 Edge 浏览器打开认证入口(https://portal.csu.edu.cn/)。

1. 在本地新建文本文档，重命名为 `网络检查.bat`。
2. 写入以下代码并保存：

```bat
@echo off
:: 向阿里云的公共网络地址发送两次测试信号
ping -n 2 223.5.5.5 >nul

:: 如果信号发送失败（代表断网了或被注销了）
if errorlevel 1 (
    :: 强制使用Edge浏览器打开你的登录网页
    start msedge "[https://portal.csu.edu.cn/](https://portal.csu.edu.cn/)"
)
